# Usability test

## Protocol (per roadmap Phase H)

5 people, no instructions, cold start at the login screen. Task: "open a
music or video app." Pass = 4/5 succeed within 60 s. Record each person's
time and what they clicked first (reveals which affordance they found, or
didn't).

**Status: not yet run.** This requires real, uninstructed testers and
cannot be simulated — it has not been done as of 2026-07-06. Run this before
submission; if fewer than 4/5 pass within 60 s, the roadmap's own guidance is
that the fix is almost always a bigger search bar or a clearer dock/menu
button, not a redesign.

## What has been verified instead (not a substitute for the above)

Manual feature walkthrough during development confirmed each surface is
reachable and functional: dock launch/focus/pin,
menu search + category browse + app launch, widget add/remove via the Widget
Store, light/dark theme switch, first-run wizard. These confirm the features
*work*, not that a first-time user *finds* them without help — that's what
the 5-person test is for.
